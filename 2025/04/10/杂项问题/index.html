1.启动docker容器报错

root@starnet:~# docker start 6ff922995a27
Error response from daemon: network 31ca337d526ca9c3aa7c9bb986f5eb770f6845b0965a987b1e3d3c6417390241 not found
Error: failed to start containers: 6ff922995a27
解决：
docker network ls
docker inspect 04d9437857ae（容器id）查看以下字段：
“Networks”: { “fl_android_90_aml_default”: {
docker network create fl_android_90_aml_default
docker network disconnect 99be520148f7（新建network的id） 04d9437857ae
docker network connect 99be520148f7 04d9437857ae
然后再重新docker start

2.System.loadLibrary方法找不到库报错

报错：java.lang.UnsatisfiedLinkError: dlopen failed: library “/system/app/hy_sign_20221201.apk!/lib/armeabi-v7a/libshell-super.2019.so” not found
对于这个方法 System.loadLibrary(“shell-super.2019”);
在4.4中优先查找/system/lib，然后再去/vendor/lib
在8.0以上的版本中，不会直接去/system/lib中查找，每个应用都有独立的 native 库搜索路径
这些路径是由链接器在加载时指定的，顺序为：
1./data/data//lib/ （一般来说这个是一个软链接，链接到/system/lib/目录）
如果apk不是直接预置在system下，那么这个软链接则会链接到/data/app/xxxx/lib/arm目录，手动安装时提取的so就会被放到这里
2.apk预置的位置，比如预置在/system/app，就会直接去apk内部寻找 ：/system/app/.apk!/lib/ （注：如果apk内的so库未进行4字节边缘对齐，则会无法识别 报错not found）
zipalign -c -v 4 demo_sign.apk 检查apk 4字节对齐情况
zipalign -v -p 4 demo_sign.apk demo_sign_align.apk 手动对齐
参考文章 https://juejin.cn/post/7455913880767463478

5.Android9.0开始系统默认禁止使用 Cleartext HTTP

Android9.0开始系统默认禁止使用 Cleartext HTTP（明文 HTTP） 流量，即直接通过 http:// 协议的非加密通信
1.可以在 AndroidManifest.xml 中设置全局允许明文流量：android:usesCleartextTraffic=”true”
2.创建安全配置文件： 在应用的 res/xml 目录下创建一个文件，例如 network_security_config.xml

修改 AndroidManifest.xml 文件： 在 标签中引用这个配置文件：android:networkSecurityConfig=”@xml/network_security_config”
